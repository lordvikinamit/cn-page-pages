# cn-page-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面来源于不同时期、不同场景下的静态内容整理，包含但不限于信息页、工具页、展示页等。仓库本身不针对任何特定域名或网站，仅作为页面资源的一个集中存放与版本管理场所。

## 项目简介

cn-page-pages 是一个纯静态 HTML 页面的归档仓库。所有页面均为独立文件，不依赖后端服务或数据库。仓库的主要目的在于：

- 集中管理分散的 HTML 页面资源
- 提供页面历史版本的可追溯性
- 便于通过 GitHub Pages 或其他静态托管服务进行发布

## 目录说明

```
cn-page-pages/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- `pages/`：每个 HTML 页面直接放置在此目录下，文件名通常反映页面用途或内容主题。
- `assets/`：存放页面共用的样式、脚本或图片资源，按需组织子目录。

## 页面归档说明

- 每个页面均为完整独立的 HTML 文件，可直接在浏览器中打开。
- 页面之间无相互依赖关系，可单独使用或发布。
- 归档时尽量保持原始结构，必要时进行链接修正或样式调整。
- 页面内容可能随时间更新，历史版本可通过 Git 提交记录查阅。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面或改进现有页面。
- 新增页面请放置在 `pages/` 目录下，并确保文件名无空格或特殊字符。
- 如需引用公共资源，请优先使用 `assets/` 中的文件。
- 不建议在页面中包含外部不可控的第三方资源链接。
- 如有疑问或建议，请通过 Issues 进行讨论。

## 许可

本仓库内容仅供学习与参考使用。具体页面的版权归属请以原始来源为准，本仓库仅作归档与展示。
